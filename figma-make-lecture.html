💬 AI 채팅창에 직접 요청 — 가장 쉬운 방법

애니메이션 동작을 말로 설명하면 Make가 CSS @keyframes 코드를 자동 작성합니다.

실습용 프롬프트 예시
헤더 텍스트가 위에서 아래로 0.6초 ease-out으로 페이드인 되게 해줘
버튼에 hover 시 scale(1.05)로 커지고 배경색이 바뀌는 효과 추가
카드 컴포넌트 3개가 0.2초 간격으로 순서대로 아래서 위로 나타나게 stagger animation 적용
로딩 화면에 spinning circle loader 만들어줘 (CSS animation만 사용)
스크롤 시 섹션이 왼쪽/오른쪽에서 교차로 슬라이드인 되는 효과 넣어줘
💡 더 정확한 결과를 원한다면: "0.5초 ease-in-out으로 opacity 0에서 1로 fadeIn" 처럼 시간 + easing + 속성 + 방향을 구체적으로 명시하세요!
🎯 Edit 도구로 특정 요소만 수정 — 정밀 수정에 좋아요
1
Edit 도구 선택

Make 미리보기 상단 툴바에서 연필(✏️) 아이콘 클릭

2
요소 직접 클릭

미리보기에서 애니메이션을 추가할 버튼, 카드, 텍스트 등을 직접 선택

3
해당 요소에만 적용될 내용 입력

예: "이 버튼에만 클릭 시 0.3초 spin animation 추가해줘"

4
Make가 선택 요소의 CSS만 타겟하여 수정

다른 요소에는 영향 없이 클릭한 요소에만 애니메이션이 적용됩니다

ℹ️ Edit 도구에서는 폰트 스타일, 텍스트 패딩도 코드 없이 직접 조작 가능합니다
⌨️ CSS 코드를 직접 붙여넣어 적용 — 개발자 방식

원하는 CSS 코드를 작성한 후 Make 채팅창에 붙여넣으면 정확하게 적용됩니다.

CSS /* 아래 CSS를 hero 타이틀에 적용해줘 */ @keyframes fadeInDown { from { opacity: 0; transform: translateY(-20px); } to { opacity: 1; transform: translateY(0); } } .hero-title { animation: fadeInDown 0.6s ease-out forwards; }
CSS · Hover 효과 /* 버튼 hover 시 위로 살짝 올라오는 효과 */ .cta-button { transition: transform 0.2s ease, box-shadow 0.2s ease; } .cta-button:hover { transform: translateY(-3px); box-shadow: 0 8px 24px rgba(124,106,255,0.3); }
💡 코드를 복사할 때 "이 코드를 .hero-title 클래스에 적용해줘"처럼 어디에 적용할지 함께 알려주면 더 정확하게 처리됩니다
04 · 작업 흐름

디자인 연결 · 수정 방법

기존 Figma 디자인을 Make에 가져오고 AI로 수정하는 방법

⚠️ URL 붙여넣기는 작동 안 해요! Figma Design 파일의 URL을 Make 채팅창에 넣으면 제대로 인식하지 못합니다. 아래 방법을 사용하세요.
📥 방법 A — 프레임 복사 · 붙여넣기
1
Figma Design 파일에서 원하는 프레임(Frame) 또는 컴포넌트 선택 (전체 페이지보다 섹션별 분리 권장)
2
Ctrl+C로 복사 후 Make AI 채팅 입력창에 Ctrl+V로 붙여넣기
3
"이 디자인을 기반으로 인터랙티브 웹페이지로 만들어줘" 같이 지시 입력
📎 방법 B — 첨부(Attach) 기능 사용
1
Make 채팅창 하단 첨부 아이콘(📎 클립) 클릭
2
Figma 파일에서 원하는 프레임 선택 후 첨부
3
프롬프트와 함께 전송 → AI가 디자인을 분석해 코드 생성

✏️ 생성된 결과물 수정하기

🎯 Edit 도구로 직접 조작

미리보기에서 요소를 클릭하면 폰트, 패딩을 코드 없이 직접 조작 가능합니다.

  • 텍스트 패딩 드래그 조절
  • 특정 요소만 타겟 수정 가능
  • 배경색 · 보더 등 시각적 속성
⌨️ 코드 탭에서 직접 확인 · 수정
1

2
코드를 복사 → VS Code 등 에디터에서 직접 수정 후 다시 붙여넣기

06 · Claude MCP 연결

Claude와 Figma Make 연결하기

MCP(Model Context Protocol)로 Claude가 Figma를 직접 읽고 제안하게 만드는 방법

🔗 MCP란?

MCP(Model Context Protocol)는 AI 모델이 외부 도구·파일·서비스에
직접 접근할 수 있게 해주는 연결 프로토콜입니다

Claude AI
↔
MCP 서버
↔
Figma API
↔
내 Figma 파일

Claude가 내 Figma 파일의 레이어 구조, 색상, 컴포넌트를 직접 읽고 분석할 수 있습니다

MCP 연결 시 무엇이 가능한가요?
✅ 연결 후 가능한 것
  • Claude가 Figma 파일 구조 직접 분석
  • 레이어명 · 색상 · 폰트 자동 추출
  • 디자인 → 코드 변환 정확도 향상
  • "이 컴포넌트 어떻게 구현할까?" 질문 가능
  • 컴포넌트 개선 제안 받기
🔧 MCP가 사용하는 Figma API
  • 파일 읽기 (레이어 트리)
  • 컴포넌트 · 스타일 조회
  • 이미지 · 에셋 추출
  • 코멘트 읽기 · 작성
  • 파일 메타데이터 조회
🛒 준비해야 할 것들
1
Claude Desktop 앱 설치

claude.ai/desktop 에서 Mac 또는 Windows 버전 다운로드 · 설치
※ 웹 버전이 아닌 데스크탑 앱에서만 MCP 설정 가능

2
Figma Personal Access Token 발급

Figma → 우측 상단 프로필 아이콘 → Settings → Security → Personal access tokens → [Generate new token] → 이름 입력 후 생성 → 토큰 복사 (한 번만 보임!)

3
Node.js 설치 확인

터미널에서 node -v 입력 → 버전 나오면 OK. 없으면 nodejs.org에서 설치

⚠️ Personal Access Token은 한 번만 표시됩니다! 반드시 메모장 등에 저장해 두세요. 잃어버리면 새로 발급해야 합니다.
⚙️ Claude Desktop MCP 설정 방법
Step 1 · 설정 파일 열기
Terminal # Mac의 경우 open ~/Library/Application\ Support/Claude/ # Windows의 경우 %APPDATA%\Claude\

해당 폴더에서 claude_desktop_config.json 파일을 텍스트 에디터로 열기

Step 2 · Figma MCP 설정 추가
claude_desktop_config.json { "mcpServers": { "figma": { "command": "npx", "args": [ "-y", "@modelcontextprotocol/server-figma" ], "env": { "FIGMA_API_TOKEN": "figd_여기에_발급받은_토큰_붙여넣기" } } } }
⚠️ figd_여기에_발급받은_토큰_붙여넣기 부분을 실제 발급받은 토큰으로 교체하세요
Step 3 · Claude Desktop 재시작
1
Claude Desktop 앱을 완전히 종료 후 재시작
2
새 대화 시작 → 입력창 하단에 🔧 도구 아이콘 또는 MCP 연결 표시 확인
3
"figma 연결됐어?" 라고 물어보거나 Figma 파일 URL을 붙여넣어 테스트
💡 설정이 잘 안 된다면 JSON 문법 오류가 많습니다. jsonlint.com에서 문법 검사 후 다시 시도해보세요
