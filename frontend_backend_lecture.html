🌐 웹사이트 = 파일 모음
웹사이트는 사실 파일 모음이에요.
HTML 파일, CSS 파일, 이미지 파일들이 서버 컴퓨터에 저장되어 있고, 우리가 주소(URL)를 입력하면 그 파일들을 받아오는 거예요.
📬 주소 = URL
https://www.naver.com 이게 바로 주소예요.
마치 택배 주소처럼, 내 브라우저가 이 주소로 파일을 요청하고 받아오는 거랍니다.
🏠 쉽게 비유하자면
웹사이트는 편의점이에요.
편의점 진열대(화면) = 프론트엔드, 창고·계산 시스템(뒷부분) = 백엔드. 손님은 진열대만 보지만, 실제 일은 뒤에서 훨씬 많이 일어나죠!
핵심 개념
식당으로 이해하는
프론트 vs 백엔드
웹사이트는 식당과 똑같아요. 손님이 보는 곳과 보지 못하는 곳이 있거든요.
🍽️
프론트엔드 (Front-end)
홀 (손님 공간)
손님이 직접 보고, 만지고, 경험하는 모든 것
메뉴판 → 웹사이트 화면
테이블 배치 → 레이아웃 디자인
조명, 인테리어 → 색상, 폰트, 애니메이션
직원이 주문받기 → 버튼 클릭 처리
↔
👨‍🍳
백엔드 (Back-end)
주방 (숨겨진 공간)
손님 눈에 보이지 않지만 핵심 처리가 이루어지는 곳
재료 보관 창고 → 데이터베이스
요리하는 과정 → 데이터 처리
주문서 관리 → 서버 로직
레시피 규칙 → 비즈니스 로직
🎯 핵심 포인트: 식당에서 맛있는 음식이 나오려면 홀과 주방 모두 필요하듯, 좋은 웹사이트도 프론트엔드와 백엔드가 함께 일해야 해요!
프론트엔드
눈에 보이는 모든 것
프론트엔드
프론트엔드는 브라우저(크롬, 엣지)가 처리해요. 사용자가 직접 경험하는 모든 것이에요.
HTML
🧱 뼈대
웹페이지의 구조를 만들어요.
마치 건물의 철골 뼈대처럼요.

제목, 문단, 버튼, 이미지 등 모든 요소의 위치와 종류를 정해요.
CSS
🎨 꾸미기
웹페이지를 예쁘게 만들어요.
건물의 인테리어와 외관 디자인처럼요.

색상, 크기, 폰트, 간격, 애니메이션을 담당해요.
JavaScript
⚡ 움직임 (가장 중요!)
웹페이지를 살아있게 만들어요. 건물의 자동문, 엘리베이터, 조명 제어 시스템처럼요.
버튼 클릭 반응, 데이터 전송, 실시간 업데이트 등 모든 동적인 기능을 담당해요.
🖥️ 눈에 보이는 모든 것 = 프론트엔드
여러분이 지금 보고 있는 이 강의자료 화면도 HTML + CSS + JavaScript로 만들어졌어요!
백엔드
눈에 보이지 않는 핵심
백엔드
백엔드는 서버 컴퓨터가 처리해요. 데이터를 저장하고 처리하는 두뇌 역할이에요.
서버 (Server)
💻 항상 켜있는 컴퓨터
24시간 365일 켜있는 컴퓨터예요.
우리가 요청하면 언제든지 응답해요.

네이버 서버는 전 세계 수백만 명이 동시에 접속해도 답해줘야 하거든요!
데이터베이스 (DB)
🗄️ 정보 창고
모든 데이터가 저장되는 곳이에요.
회원정보, 게시글, 주문내역, 사진 등

엑셀처럼 행과 열로 데이터를 정리해서 저장해요.
구분 프론트엔드 백엔드
위치 내 컴퓨터 브라우저 서버 컴퓨터 (멀리 있음)
볼 수 있나요? ✅ 보여요 ❌ 안 보여요
주요 언어 HTML, CSS, JavaScript Python, Java, Node.js 등
하는 일 화면 표시, 사용자 조작 데이터 처리, 저장, 보안

잘 이해했나요?
퀴즈로 확인해봐요!

Q2. 내가 쿠팡에서 주문할 때 주문 정보를 저장하고 결제를 처리하는 것은?
Q3. 다음 중 프론트엔드 기술이 아닌 것은?
배포 준비
구글 앱스크립트로
무료 백엔드 만들기
백엔드 서버를 직접 만들려면 비용이 들어요. 하지만 구글이 무료로 서버를 빌려줘요!
Google Apps Script (GAS)
구글이 서버가 되어준다!
구글 드라이브, 시트, Gmail과 연동되는 서버 코드를 작성하면, 구글이 직접 실행시켜줘요. 서버 컴퓨터 없이도 진짜 백엔드 기능을 만들 수 있어요!
✅ 완전 무료 ✅ 구글 계정만 있으면 OK ✅ JavaScript로 작성 ✅ URL로 배포 가능
🤔 일반 백엔드 (어려움)
서버 컴퓨터 구매/임대 → 서버 설정 → 도메인 구매 → 보안 설정 → 유지보수...

💰 월 몇 만원~몇십 만원 비용 발생
😊 구글 앱스크립트 (쉬움)
구글 드라이브 접속 → 새 파일 → 코드 작성 → 배포 버튼 클릭!

🎉 무료! 구글이 서버 관리해줌!
🚀 오늘 목표: 구글 앱스크립트로 폼 데이터를 받아서 구글 시트에 저장하는 웹앱을 배포해봐요!
실습 가이드
구글 앱스크립트
배포 단계별 가이드
따라하면 누구나 할 수 있어요! 차근차근 진행해봐요.
1
🗂️ 구글 시트 만들기
구글 드라이브에서 새 구글 스프레드시트를 만들어요. 데이터가 저장될 창고예요. A1셀에 "이름", B1셀에 "이메일" 등 헤더를 적어두세요.
2
📝 앱스크립트 에디터 열기
구글 시트 상단 메뉴에서 확장 프로그램 → Apps Script 클릭. 코드 에디터 창이 열려요. 여기가 백엔드 코드를 작성하는 곳이에요.
// 이 함수는 폼에서 데이터가 올 때 자동 실행돼요 function doPost(e) { // 현재 스프레드시트 열기 const sheet = SpreadsheetApp .getActiveSpreadsheet() .getActiveSheet(); // 폼에서 받은 데이터 꺼내기 const name = e.parameter.name; const email = e.parameter.email; // 시트에 새 행 추가하기 sheet.appendRow([name, email, new Date()]); // 완료 응답 보내기 return ContentService .createTextOutput('저장 완료!'); }
3
🚀 웹앱으로 배포하기
오른쪽 상단 배포 → 새 배포 클릭. 유형: 웹 앱 선택. "다음 사용자로 실행": 나. "액세스 권한": 모든 사용자 선택 후 배포!
4
🔗 URL 복사하기
배포 후 웹 앱 URL이 생성돼요. 이게 바로 우리의 백엔드 주소예요!
https://script.google.com/macros/s/ABC123XYZ.../exec
이 URL로 데이터를 보내면 구글 시트에 자동 저장돼요!
5
🖥️ 프론트엔드(HTML) 연결하기
이제 HTML 폼에서 위 URL로 데이터를 전송하면 끝!
<!-- HTML 폼 예시 --> <form id="myForm"> <input name="name" placeholder="이름"> <input name="email" placeholder="이메일"> <button>제출</button> </form> <script> const GAS_URL = '여기에 배포 URL 붙여넣기'; document.getElementById('myForm') .addEventListener('submit', (e) => { e.preventDefault(); const data = new FormData(e.target); fetch(GAS_URL, { method: 'POST', body: data }).then(() => alert('✅ 제출 완료!')); }); </script>
✓
🎉 완성! 프론트 + 백엔드가 연결됐어요
HTML 파일 = 프론트엔드 (사용자가 보는 화면)
구글 앱스크립트 = 백엔드 (데이터 처리)
구글 시트 = 데이터베이스 (데이터 저장)

이제 여러분은 프론트엔드부터 백엔드까지 갖춘 풀스택 웹앱을 만든 거예요! 🚀
