Firebase란 무엇인가요?
앱 개발의 뒷부분을 대신 해주는 마법 같은 도구

🏢
Firebase = 구글이 만든 앱 개발 플랫폼

앱을 만들 때 회원가입, 데이터 저장, 알림 발송 같은 기능은 서버(백엔드)가 필요해요. Firebase는 이 서버 역할을 대신 해줘서, 개발자가 서버 없이도 앱을 만들 수 있게 해줍니다.

💡 쉬운 비유: 식당에서 직접 주방을 차릴 필요 없이 외식하는 것처럼, 서버를 직접 구축하지 않고 Firebase의 서버를 빌려 쓰는 개념이에요!

🚫 Firebase 없이 개발할 때

  • 서버 컴퓨터 직접 구축
  • 서버 코드 직접 작성
  • 데이터베이스 설치/관리
  • 보안 설정 직접 처리
  • 서버 유지보수 필요

✨ Firebase 사용하면

  • 서버 없이 바로 시작
  • 몇 줄 코드로 기능 구현
  • 데이터베이스 자동 제공
  • 보안 기본 내장
  • 구글이 대신 관리

📱
어떤 앱에서 Firebase를 쓸까요?

  • 💬 채팅 앱 (카카오톡 같은 메시징 기능)
  • 🛒 쇼핑 앱 (상품 목록, 주문 데이터 저장)
  • 📸 SNS 앱 (사진 업로드, 좋아요 기능)
  • 🎮 게임 앱 (점수 저장, 순위표)
  • 📚 교육 앱 (학습 기록, 진도 저장)

💰
가격 정책 — 무료로 충분해요!

Firebase는 Spark 플랜(무료)으로 시작할 수 있어요. 소규모 프로젝트는 무료 한도 내에서 충분히 사용 가능해요. 사용량이 늘면 Blaze(종량제) 플랜으로 업그레이드할 수 있어요.

💡 학습용, 포트폴리오, 소규모 프로젝트는 무료 플랜으로 충분해요!

🗂️ Firestore 데이터 구조 이해하기

Firestore는 폴더 구조처럼 데이터를 관리해요:

// 컬렉션(Collection) → 문서(Document) → 필드(Field) users (컬렉션) └── user001 (문서 ID) ├── name: "홍길동" ├── age: 25 └── email: "hong@example.com" └── user002 (문서 ID) ├── name: "김철수" └── age: 30
💡 컬렉션 = 폴더, 문서 = 파일, 필드 = 파일 안의 내용 으로 생각하면 쉬워요!

6

설정 코드(firebaseConfig) 복사

아래와 같은 코드가 나타나요. 이걸 복사해서 프로젝트에 붙여넣어요!

⚙️ 내 프로젝트의 설정 코드 예시

const firebaseConfig = { apiKey: "AIzaSyC1234abcd...", // API 키 authDomain: "my-app.firebaseapp.com", // 인증 도메인 projectId: "my-app", // 프로젝트 ID storageBucket: "my-app.appspot.com", // 스토리지 버킷 messagingSenderId: "123456789", // 메시지 발신자 ID appId: "1:123456:web:abcdef..." // 앱 ID };
⚠️ 주의! 이 설정 코드에는 API 키가 포함되어 있어요. GitHub에 올릴 때는 환경변수(.env 파일)로 관리하는 게 보안상 안전해요!
4
코드 실습: Firestore 데이터 다루기
데이터를 읽고, 쓰고, 수정하고, 삭제하는 기초 코드

📦
Firebase SDK 설치

터미널(명령 프롬프트)에서 아래 명령어를 실행해요:

npm install firebase

⚙️
1단계: Firebase 초기화 (firebase.js)

// firebase.js — 프로젝트의 Firebase 설정 파일 import { initializeApp } from 'firebase/app'; import { getFirestore } from 'firebase/firestore'; const firebaseConfig = { apiKey: "여기에-API-KEY", authDomain: "프로젝트명.firebaseapp.com", projectId: "프로젝트명", }; const app = initializeApp(firebaseConfig); export const db = getFirestore(app); // ↑ 이 db를 다른 파일에서 불러와서 사용해요

✍️
2단계: 데이터 추가하기 (Create)

import { collection, addDoc } from 'firebase/firestore'; import { db } from './firebase'; const addUser = async () => { await addDoc(collection(db, 'users'), { name: '홍길동', age: 25, email: 'hong@example.com', createdAt: new Date() }); console.log('저장 완료! 🎉'); };
💡 addDoc()은 문서 ID를 Firebase가 자동으로 생성해줘요!

📖
3단계: 데이터 읽기 (Read)

import { collection, getDocs } from 'firebase/firestore'; const getUsers = async () => { const querySnapshot = await getDocs(collection(db, 'users')); querySnapshot.forEach((doc) => { console.log('ID:', doc.id); console.log('데이터:', doc.data()); }); };

✏️
4단계: 데이터 수정하기 (Update)

import { doc, updateDoc } from 'firebase/firestore'; const updateUser = async () => { const userRef = doc(db, 'users', '문서ID'); await updateDoc(userRef, { age: 26 // age 필드만 수정 }); console.log('수정 완료!'); };

⚡
5단계: 실시간 감지 (onSnapshot)

import { collection, onSnapshot } from 'firebase/firestore'; // 데이터가 바뀔 때마다 자동으로 실행돼요! onSnapshot(collection(db, 'users'), (snapshot) => { snapshot.docs.forEach((doc) => { console.log('실시간 업데이트:', doc.data()); }); });
💡 채팅앱처럼 메시지가 실시간으로 나타나게 하려면 onSnapshot을 써요! getDocs는 한 번만 읽고, onSnapshot은 계속 감지해요.

🗑️ 데이터 삭제하기 (Delete)

import { doc, deleteDoc } from 'firebase/firestore'; const deleteUser = async () => { await deleteDoc(doc(db, 'users', '문서ID')); console.log('삭제 완료!'); };

📌 CRUD 요약표

작업 함수 설명
➕ 추가addDoc()새 문서 추가
📖 읽기getDocs()문서 목록 한 번 읽기
⚡ 실시간onSnapshot()변화 감지 + 자동 업데이트
✏️ 수정updateDoc()기존 문서 일부 수정
🗑️ 삭제deleteDoc()문서 삭제
