01 Web vs App

웹(Web)과 앱(App),
무엇이 다를까?

두 개념을 정확하게 이해하는 것이 시작입니다

🌐

웹 (Web)

인터넷 브라우저(크롬, 사파리, 엣지 등)를 통해 접속하는 디지털 공간입니다. 주소창에 URL을 입력하면 바로 접근할 수 있어요.

설치 불필요 URL로 접근 모든 기기에서 접속
📱

앱 (App / Application)

기기에 직접 설치하여 사용하는 소프트웨어입니다. 앱스토어나 플레이스토어에서 다운로드해야 사용할 수 있어요.

스마트폰에 설치 앱스토어 배포 기기 기능(카메라 등) 활용
비교 항목 🌐 웹 📱 앱
접근 방법 브라우저 주소창에 URL 입력 앱스토어에서 설치 후 아이콘 클릭
업데이트 서버에서 자동 업데이트 (사용자 불필요) 사용자가 직접 업데이트 다운로드
개발 난이도 상대적으로 진입 장벽이 낮음 iOS/Android 플랫폼별 별도 개발 필요
속도 인터넷 환경에 따라 달라짐 기기에 설치되어 상대적으로 빠름
기기 기능 활용 제한적 (최근 개선되는 중) 카메라, GPS, 알림 등 완전 활용
예시 네이버 뉴스, 유튜브 웹 카카오톡, 인스타그램 앱
02 What is a Web App

그렇다면 웹앱(Web App)은?

웹과 앱, 두 가지 장점을 결합한 새로운 형태

⚡

웹앱 = 웹 기술로 만들어졌지만, 앱처럼 동작하는 서비스

브라우저에서 실행되지만 화면 전환, 실시간 데이터 업데이트, 복잡한 사용자 상호작용이 가능한 웹 서비스입니다. 과거 웹이 단순히 정보를 보여주는 역할에 그쳤다면, 웹앱은 사용자가 직접 작업을 수행하고 결과를 저장·공유할 수 있습니다.

📧

Gmail

설치 없이 브라우저만으로 이메일 작성, 첨부파일, 검색, 라벨 관리까지 완전한 이메일 앱 경험

📊

Google 스프레드시트

웹에서 엑셀처럼 표를 편집하고, 여러 명이 동시에 작업하는 실시간 협업 앱

🎵

Spotify Web Player

설치 없이 브라우저에서 음악을 검색하고 재생 목록을 관리하는 스트리밍 앱

💬

Slack (웹 버전)

브라우저에서 팀 채팅, 파일 공유, 화상회의 링크까지 연결되는 협업 플랫폼

🎨

Figma

웹브라우저에서 UI 디자인 작업을 하고 팀원과 실시간으로 피드백을 주고받는 디자인 도구

🛒

쿠팡 / 배민

상품 검색, 장바구니, 결제, 주문 관리까지 모두 웹 기술로 구현된 쇼핑·주문 앱

03 Types of Web

웹의 종류

시간 순서대로 보면 왜 웹앱이 등장했는지 이해됩니다

1
1세대 — 1990년대

정적 웹사이트 (Static Website)

HTML 파일 자체가 콘텐츠입니다. 누가 접속해도 같은 화면이 보이고, 내용을 바꾸려면 개발자가 직접 파일을 수정해야 합니다. 로그인, 검색, 장바구니 같은 기능은 없어요.

예시: 회사 소개 페이지, 이력서 사이트, 단순 포트폴리오
2
2세대 — 2000년대 초반

동적 웹사이트 (Dynamic Website)

서버가 사용자의 요청에 따라 다른 페이지를 생성해서 보여줍니다. 로그인 기능, 검색 결과, 게시판 댓글 등이 가능해졌어요. 하지만 버튼 하나 클릭해도 페이지 전체가 새로 로딩됩니다.

예시: 초기 네이버·다음 포털, 게시판 서비스, 초창기 쇼핑몰
3
3세대 — 2010년대

웹앱 / SPA (Single Page Application)

페이지 전체를 새로 불러오지 않고, 필요한 부분만 바뀝니다. 화면 전환이 부드럽고 앱처럼 빠릅니다. 데이터는 API를 통해 실시간으로 주고받아요. React, Vue, Angular 같은 프레임워크로 개발합니다.

예시: Gmail, 트위터/X, 카카오페이, 쿠팡
4
최신 — 2020년대

PWA (Progressive Web App)

웹이지만 스마트폰 홈 화면에 추가하고, 인터넷이 없어도 일부 기능을 사용할 수 있고, 푸시 알림도 받을 수 있습니다. 앱스토어 없이 앱 수준의 경험을 제공합니다.

예시: 트위터 라이트, 스타벅스 웹앱, 핀터레스트 PWA
04 Web App Architecture

웹앱의 구조

웹앱은 3가지 층(Layer)으로 나뉩니다 — 레스토랑에 비유해볼게요 🍽️

🖥️
LAYER 01 — 프론트엔드 (Front-End)
홀 — 손님이 보고 경험하는 공간
사용자가 실제로 눈으로 보고 클릭하며 상호작용하는 화면입니다. 브라우저에서 직접 실행됩니다. 디자인, 버튼, 입력창, 화면 전환이 모두 이 영역입니다.
HTML — 메뉴판 구조 CSS — 인테리어·디자인 JavaScript — 직원의 행동 React / Vue — 효율적인 주문 시스템 Figma (디자인 도구)
⇅ API 통신 (주문서 전달 / 음식 수령)
⚙️
LAYER 02 — 백엔드 (Back-End)
주방 — 실제 요리가 일어나는 공간
사용자 눈에는 보이지 않지만 핵심 비즈니스 로직이 여기서 처리됩니다. 로그인 확인, 결제 처리, 데이터 계산 등 중요한 작업이 모두 서버에서 이루어집니다.
Node.js Python (Django / Flask) Java (Spring) PHP API 설계 (REST / GraphQL) 서버 보안 / 인증
⇅ 데이터 요청 / 저장 (재료 꺼내기 / 넣기)
🗄️
LAYER 03 — 데이터베이스 (Database)
창고 — 모든 재료(데이터)를 보관하는 공간
회원 정보, 게시글, 주문 내역, 상품 정보 등 모든 데이터를 체계적으로 저장하고 필요할 때 꺼내주는 곳입니다.
MySQL · PostgreSQL (정형 데이터) MongoDB (비정형 / 유연한 구조) Firebase (실시간 · 초보자 추천) Redis (빠른 임시 저장소)
💡
정리하면: 손님(사용자)이 웹브라우저(홀)에서 주문(클릭)을 하면 → 프론트엔드가 주문서(API 요청)를 만들어 백엔드(주방)로 보내고 → 백엔드가 창고(데이터베이스)에서 필요한 재료(데이터)를 꺼내 요리(처리)해서 → 다시 프론트엔드(홀)를 통해 손님에게 결과를 보여줍니다.

01

Header (헤더)

페이지 최상단 영역

사이트 로고, 로그인/로그아웃 버튼, 검색창, 장바구니 아이콘 등이 위치합니다. 어떤 페이지를 가도 고정되어 항상 보이는 경우가 많아요.

로고 검색창 로그인 버튼
02

Navigation (네비게이션)

메뉴 링크 모음

사이트 내 각 페이지로 이동하는 메뉴 링크 모음입니다. 상단 가로 메뉴(GNB), 왼쪽 세로 메뉴(사이드바), 모바일 햄버거 메뉴 형태가 있습니다.

GNB 사이드바 햄버거 메뉴
03

Hero / Banner (히어로)

메인 대표 화면

페이지에서 가장 크고 눈에 띄는 상단 영역입니다. 서비스의 핵심 메시지, 배경 이미지, 그리고 행동 유도 버튼(CTA)을 담아 첫인상을 결정합니다.

대표 문구 CTA 버튼 배경 이미지
04

Button (버튼)

사용자 행동 유도

클릭하면 특정 동작이 실행되는 요소입니다. 제출(Submit), 취소(Cancel), 더보기, 구매하기 등 사용자가 원하는 행동을 유도할 때 사용합니다.

Primary 버튼 Secondary 버튼 아이콘 버튼
05

Form (폼 / 양식)

사용자 입력 영역

사용자가 정보를 직접 입력하는 영역입니다. 로그인, 회원가입, 검색창, 댓글 작성, 주문서 작성이 모두 폼입니다. 입력창(input), 체크박스, 라디오 버튼 등을 포함합니다.

로그인 폼 검색창 회원가입
06

Card (카드)

정보를 묶어 표현

관련 정보를 네모 박스 형태로 묶어 보여주는 컴포넌트입니다. 상품 카드, 뉴스 카드, 프로필 카드처럼 반복되는 정보를 일관되게 표시할 때 사용합니다.

상품 카드 뉴스 카드 프로필 카드
07

Modal (모달 / 팝업)

화면 위에 뜨는 창

현재 페이지 위에 겹쳐서 나타나는 창입니다. 알림, 확인 요청, 상세 정보 보기, 이미지 확대 등 흐름을 방해하지 않고 추가 정보를 전달할 때 씁니다.

확인 팝업 이미지 뷰어 알림창
08

Footer (푸터)

페이지 최하단 영역

페이지 맨 아래에 위치하며 저작권 정보, 회사 정보, 이용약관·개인정보처리방침 링크, 사이트맵, SNS 링크 등 보조 정보를 담습니다.

저작권 이용약관 사이트맵
+

06 Development Process

웹앱 개발 과정

아이디어에서 실제 서비스까지, 어떤 단계를 거칠까?

STEP 01

📋 기획

어떤 문제를 해결할 서비스인지 결정합니다. 어떤 기능이 필요한지 목록(기능 명세서)을 작성하고, 누가 사용할지(타깃 사용자)를 정합니다.

STEP 02

🎨 UI/UX 디자인

화면 구성을 설계합니다. 스케치(와이어프레임) → 상세 디자인(목업) → 클릭 가능한 시제품(프로토타입) 순서로 진행합니다.

STEP 03

🖥️ 프론트엔드 개발

HTML, CSS, JavaScript로 사용자 화면을 구현합니다. 디자인 시안을 보며 코드로 화면을 만들고, API 연결을 준비합니다.

STEP 04

⚙️ 백엔드 개발

서버와 API를 구성합니다. 로그인, 회원가입, 데이터 저장 등 비즈니스 로직과 보안 처리를 담당합니다.

STEP 05

🗄️ 데이터베이스 설계

어떤 데이터를 어떤 형태로 저장할지 구조를 설계합니다. 회원 테이블, 게시글 테이블 등 관계를 정리하고 구축합니다.

STEP 06

🚀 테스트 & 배포

오류를 찾아 수정(QA)한 후, 실제 서버에 올려(배포) 모든 사람이 접속할 수 있게 합니다. 운영 후에도 지속적으로 개선합니다.

🌟

오늘 강의 핵심 정리

① 웹(브라우저 접속)과 앱(설치 실행)은 다르지만, 웹앱은 두 장점을 결합한 서비스입니다.
② 웹의 종류는 정적→동적→웹앱(SPA)→PWA 순서로 발전했습니다.
③ 웹앱의 구조는 프론트엔드(화면) + 백엔드(처리) + 데이터베이스(저장) 3층으로 이루어집니다.
④ 6가지 필수 요소(HTML·CSS·JS·서버·DB·API)가 함께 작동해야 완전한 웹앱이 됩니다.

웹앱 구조와 필수요소 — 강의자료

HTML · CSS · JS · SERVER · DB · API
